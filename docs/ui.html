What the numbers mean

theta_norm
The L2 norm of the weight vector θ for each arm. Higher means the arm has a stronger learned signal — the model is more confident about when to recommend it.
prediction_count / reward_count
How many times each arm has been selected, and how many of those received a reward. A gap means those interactions are still in-flight — reward not yet sent.
alpha
Exploration parameter. Higher = more exploration. LinUCB adds a larger uncertainty bonus; Thompson Sampling samples from a wider posterior. Default 1.0.
Context vector
A fixed-length array of floats in [0, 1] describing the current situation. BanditDB uses this to personalise decisions — the model learns which arms work best for which context profiles.
