// how it works

Three API calls. That's the entire integration.

The same loop whether you're routing LLM tasks, assigning treatments, or picking offers.

01 Predict

Pass a context vector. Get back the optimal arm and an interaction ID for the feedback loop.

arm, iid = db.predict(
  "campaign",
  context,
)
02 Act

An agent routes to the best model. A patient gets the most promising treatment. A customer sees the winning offer.

# agent succeeds
result = run_task(arm, prompt)

# patient responds
outcome = treat(patient, arm)

# customer converts
revenue = show_offer(user, arm)
03 Reward

Report the outcome. A⁻¹ and θ update in microseconds via Sherman-Morrison. The next request is smarter.

db.reward(
  iid,
  1.0 if success else 0.0,
)

// quick start · llm routing

AI Agent LLM Routing

AI agents blindly routing every task to GPT-4o spend 60× more than necessary. BanditDB learns — from every agent in your swarm — exactly which model wins for which type of task.

  • ▸ every agent shares the same learned intuition
  • ▸ converges to optimal routing without configuration
  • ▸ works for pricing, personalisation, clinical trials
agent.py 
from banditdb import Client

db = Client("http://localhost:8080",
            api_key="your-secret-key")

# register the campaign once
db.create_campaign("llm_routing",
    arms=["gpt-4o", "claude-haiku", "llama-3"],
    feature_dim=3)

# for each task
context = [
    prompt_length / 1000,    # complexity
    has_math * 1.0,          # reasoning load
    budget_remaining / 100,  # cost pressure
]
model, iid = db.predict("llm_routing", context)
result = run_task(model, prompt)
db.reward(iid, 1.0 if result.success else 0.0)
# the entire swarm learns from this interaction

// algorithms

Two algorithms. One API.

Pick at campaign creation. Both share identical per-arm state (A⁻¹, b, θ) — the predict → reward loop is unchanged.

UCB
LinUCB
algorithm="linucb"  ·  default

Deterministic upper confidence bound. Exploration is fully tunable via alpha. Sweep it offline once to calibrate exactly how long to keep exploring.

score = θ·x + α·√(x·A⁻¹·x)
TS
Linear Thompson Sampling
algorithm="thompson_sampling"

Samples θ̃ from the Bayesian posterior on every call. No alpha sweep needed — alpha=1.0 is the natural posterior width. Concurrent users automatically diversify coverage.

θ̃ ~ N(θ, α²·A⁻¹)   score = θ̃·x
campaigns.py 
# LinUCB — tune alpha to dial exploration up or down
db.create_campaign("ucb_offers", arms=["a", "b", "c"], feature_dim=4, alpha=1.5)

# Thompson Sampling — Bayesian exploration, works out of the box
db.create_campaign("ts_offers",  arms=["a", "b", "c"], feature_dim=4,
                   algorithm="thompson_sampling")
