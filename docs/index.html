How it works

Four API calls — one to define, three to learn.

1

Create

run once at startup

Name the campaign, list the arms, and set the context dimension. BanditDB initialises the weight matrices and is immediately ready to serve predictions.

db.create_campaign(
    "sleep",
    arms=[
        "decrease_temperature",
        "decrease_light",
        "decrease_noise",
    ],
    feature_dim=5,
)
↻ loops with every participant
2

Predict

Pass the participant's context vector. Get back the recommended intervention and an interaction ID to track it.

context = [
    1.0,   # female
    0.35,  # age 35
    0.50,  # 75 kg
    0.60,  # active
    0.96,  # bedtime 23:00
]

arm, iid = db.predict("sleep", context)
# → "decrease_temperature"
3

Act

Apply the chosen intervention. BanditDB holds the context in its TTL cache, ready to receive the reward when the outcome is known.

# arm = "decrease_temperature"
apply_intervention(user_id, arm)

# lower bedroom temperature to 17°C
# BanditDB waits for the outcome...
4

Reward

Report the outcome the next morning. Matrices update in microseconds. Every subsequent participant gets a smarter recommendation.

# next morning: PSQI scores
score_before = 62
score_after  = 79

reward = (score_after - score_before) / score_before
# reward = 0.27

db.reward(iid, reward)
Quick Start

AI Agent LLM Routing

AI agents blindly routing every task to GPT-4o spend 60× more than necessary. BanditDB learns — from every agent in your swarm — exactly which model wins for which type of task.

→ Every agent shares the same learned intuition
→ Converges to optimal routing without configuration
→ Works for pricing, personalisation, clinical protocols — any decision with measurable outcomes 
from banditdb import Client

db = Client("http://localhost:8080",
            api_key="your-secret-key")

db.create_campaign(
    "llm_routing",
    arms=["gpt-4o", "claude-haiku", "llama-3"],
    feature_dim=3,
)

context = [
    prompt_length / 1000,
    has_math * 1.0,
    budget_remaining / 100,
]

model, iid = db.predict("llm_routing", context)

result = run_task(model, prompt)
db.reward(iid, 1.0 if result.success else 0.0)

Two algorithms. One API.

Pick the algorithm at campaign creation. Everything else stays identical.

UCB
LinUCB
algorithm="linucb"
Default

Adds a deterministic upper confidence bound to every arm score. Exploration is tunable via alpha — sweep it offline to calibrate.

score = θ·x + α·√(x·A⁻¹·x)
TS
Linear Thompson Sampling
algorithm="thompson_sampling"

Samples θ̃ from the Bayesian posterior on every prediction. No alpha sweep needed — alpha=1.0 is the natural posterior width. Concurrent users automatically diversify arm coverage.

θ̃ ~ N(θ, α²·A⁻¹)  score = θ̃·x 
db.create_campaign("ucb_offers", arms=["a", "b", "c"], feature_dim=4, alpha=1.5)

db.create_campaign("ts_offers", arms=["a", "b", "c"], feature_dim=4,
                   algorithm="thompson_sampling")

For AI Agents

Your agent swarm builds shared intuition

Standard LLM agents are stateless — if they make a bad decision, they repeat it tomorrow. BanditDB's built-in MCP server gives your entire agent swarm shared persistent memory.

Two commands and BanditDB is a native tool in Claude. Your agents can get intuition, record outcomes, and inspect learning state — no config file editing required.

Every decision made by any agent in the network improves the routing for all future agents.

Any decision with a measurable outcome

BanditDB works wherever you can define a context, a set of choices, and a reward.
