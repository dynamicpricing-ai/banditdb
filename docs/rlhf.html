Reinforcement Learning from Human Feedback

Three-stage pipeline — math derivations + numerical examples at each step

1

Supervised Fine-Tuning (SFT)

Goal

Start from a pretrained LLM and fine-tune it on human-written demonstrations so the model learns to follow instructions.

Loss Function

Standard cross-entropy over a dataset of (prompt $x$, response $y$) pairs. The model is trained to predict each token given the prompt and all previous tokens:

$$\mathcal{L}_{\text{SFT}}(\theta) = -\sum_{t=1}^{T} \log \pi_\theta(y_t \mid x,\, y_{\lt t})$$

where $\pi_\theta$ is the LLM policy parameterised by weights $\theta$, and $y_{\lt t}$ denotes all tokens before position $t$.

Numerical Example

Example

Prompt $x$ = "What is 2 + 2?"  ·  Target response $y$ = "4 <eos>" (2 tokens)

Suppose the current model assigns:

$$\pi_\theta(\text{"4"} \mid x) = 0.8, \qquad \pi_\theta(\langle\text{eos}\rangle \mid x, \text{"4"}) = 0.9$$
$$\mathcal{L}_{\text{SFT}} = -\bigl(\log 0.8 + \log 0.9\bigr) = -(-0.223 - 0.105) = 0.328$$

After one gradient step $\theta \leftarrow \theta - \eta \nabla \mathcal{L}$, the model assigns higher probability to the correct tokens, reducing the loss.

SFT teaches the model how to follow instructions, but not which responses humans prefer. That requires Stage 2.
2

Reward Model Training

Human Preference Data

For each prompt $x$, two responses are generated. A human labels which one they prefer. The dataset is a collection of triples $(x, y_w, y_l)$ where $y_w$ is the winner and $y_l$ the loser.

Bradley-Terry Preference Model

We model the probability that a human prefers $y_w$ over $y_l$ given prompt $x$:

$$P(y_w \succ y_l \mid x) = \sigma\!\bigl(r_\phi(x, y_w) - r_\phi(x, y_l)\bigr)$$

where $r_\phi : (\text{prompt}, \text{response}) \to \mathbb{R}$ is the reward model (a neural net with parameters $\phi$) and $\sigma(z) = \frac{1}{1 + e^{-z}}$ is the sigmoid function.

Training Loss

Negative log-likelihood over all preference pairs:

$$\mathcal{L}_{\text{RM}}(\phi) = -\,\mathbb{E}_{(x,\,y_w,\,y_l)}\!\left[\log \sigma\!\bigl(r_\phi(x, y_w) - r_\phi(x, y_l)\bigr)\right]$$

Minimising this loss pushes $r_\phi(x, y_w) - r_\phi(x, y_l)$ to be large and positive for all preferred pairs.

Numerical Example

Pair A — model is confident

Prompt: "Explain gravity."

  • $y_w$: "Gravity is a force attracting masses together."
  • $y_l$: "Gravity is magic."
$$r_\phi(x, y_w) = 2.1, \quad r_\phi(x, y_l) = 0.4 \quad \Rightarrow \quad \Delta = 1.7$$
$$P(y_w \succ y_l) = \sigma(1.7) = \frac{1}{1 + e^{-1.7}} = \frac{1}{1.183} \approx 0.845$$
$$\text{Loss} = -\log(0.845) \approx 0.168 \quad \text{(small — model is correct)}$$
Pair B — model is confused
$$r_\phi(x, y_w) = 1.1, \quad r_\phi(x, y_l) = 1.0 \quad \Rightarrow \quad \Delta = 0.1$$
$$P(y_w \succ y_l) = \sigma(0.1) \approx 0.525$$
$$\text{Loss} = -\log(0.525) \approx 0.644 \quad \text{(large — model barely distinguishes the two)}$$

Gradient descent will increase $r_\phi(x, y_w)$ and decrease $r_\phi(x, y_l)$ to reduce this loss.

3

RL Fine-Tuning with PPO

The RLHF Objective

The LLM policy $\pi_\theta$ generates a response $y$ for prompt $x$. We maximise the reward while keeping the policy close to the original SFT model $\pi_\text{ref}$:

$$\mathcal{J}(\theta) = \mathbb{E}_{x \sim \mathcal{D},\; y \sim \pi_\theta(\cdot|x)}\!\left[\, r_\phi(x, y)\; -\; \beta\, \log \frac{\pi_\theta(y \mid x)}{\pi_\text{ref}(y \mid x)} \right]$$

The two terms:

  • $r_\phi(x, y)$ — reward from the reward model (maximise quality)
  • $\beta\, \log \frac{\pi_\theta}{\pi_\text{ref}}$ — KL penalty preventing the policy from drifting too far from the SFT baseline (reward hacking guard)

KL Divergence Term — Numerical Example

Example

For a single token "helpful":

$$\pi_\theta(\text{"helpful"} \mid x) = 0.9, \qquad \pi_\text{ref}(\text{"helpful"} \mid x) = 0.7$$
$$\text{KL contribution} = \log\frac{0.9}{0.7} = \log(1.286) \approx 0.251$$

With $\beta = 0.1$, the penalty on this token is $0.1 \times 0.251 = 0.025$ reward units — small because the drift is modest.

If the policy had drifted further, e.g. $\pi_\theta = 0.95$:

$$\text{KL contribution} = \log\frac{0.95}{0.7} \approx 0.304 \quad \Rightarrow \quad \text{penalty} = 0.030$$

The KL term grows nonlinearly as the policy diverges, acting as a soft constraint.

PPO Clip Objective

PPO updates the policy using a clipped surrogate loss to prevent destabilising large gradient steps:

$$\mathcal{L}_{\text{PPO}}(\theta) = \mathbb{E}_t\!\left[\min\!\left(\rho_t\, A_t,\;\; \operatorname{clip}(\rho_t,\, 1-\varepsilon,\, 1+\varepsilon)\, A_t\right)\right]$$

where:

  • $\rho_t = \dfrac{\pi_\theta(y_t \mid x)}{\pi_{\theta_\text{old}}(y_t \mid x)}$ — probability ratio between new and old policy
  • $A_t$ — advantage: how much better this response was than the baseline value estimate
  • $\varepsilon$ — clip threshold, typically $0.2$

PPO Clip — Numerical Example

Case 1 — good response, moderate ratio
$$\pi_\theta = 0.6, \quad \pi_{\theta_\text{old}} = 0.5 \quad\Rightarrow\quad \rho = 1.2, \quad A = +1.5, \quad \varepsilon = 0.2$$
$$\text{unclipped} = 1.2 \times 1.5 = 1.8$$
$$\text{clipped} = \operatorname{clip}(1.2,\, 0.8,\, 1.2) \times 1.5 = 1.2 \times 1.5 = 1.8$$

At the boundary — no penalty.

Case 2 — good response, ratio too large (clipping fires)
$$\rho = 1.4, \quad A = +1.5$$
$$\text{unclipped} = 1.4 \times 1.5 = 2.1$$
$$\text{clipped} = \operatorname{clip}(1.4,\, 0.8,\, 1.2) \times 1.5 = 1.2 \times 1.5 = 1.8 \quad\leftarrow \min \text{ is taken}$$

The clip prevents over-exploiting a lucky high-reward sample with a disproportionately large policy update.

Case 3 — bad response, ratio too small (clipping fires on negative side)
$$\rho = 0.6, \quad A = -1.0$$
$$\text{unclipped} = 0.6 \times (-1.0) = -0.6$$
$$\text{clipped} = \operatorname{clip}(0.6,\, 0.8,\, 1.2) \times (-1.0) = 0.8 \times (-1.0) = -0.8 \quad\leftarrow \min \text{ is taken}$$

The policy is penalised more conservatively for suppressing bad tokens too aggressively.

Advantage Estimation

The advantage $A_t$ is computed using Generalised Advantage Estimation (GAE). A value network $V_\psi(x)$ estimates the expected return from state $x$:

$$\delta_t = r_t + \gamma V_\psi(s_{t+1}) - V_\psi(s_t)$$
$$A_t^{\text{GAE}} = \sum_{k=0}^{\infty} (\gamma\lambda)^k\, \delta_{t+k}$$

where $\gamma \in (0,1)$ is the discount factor and $\lambda \in (0,1)$ controls the bias-variance trade-off of the estimate.
